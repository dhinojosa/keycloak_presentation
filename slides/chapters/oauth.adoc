== OAuth 2.0
image::iStock-1181695869.jpg[background, size=cover]

=== OAuth 2.0

* Authorization framework
* Delegates access without sharing passwords
* Issues Access Tokens
* Uses scopes to limit requested access
* Does not prove who the user is

[.small]
Spec: https://www.rfc-editor.org/rfc/rfc6749[RFC 6749: OAuth 2.0]

=== OAuth Is Not Authentication

* OAuth answers: "Can this client access this resource?"
* OIDC answers: "Who signed in?"
* Access Tokens are for APIs
* ID Tokens are for clients

=== OAuth Participants

[cols="35b,65"]
|===
| Participant | Role

| Resource Owner
| User or system that owns the access

| Client
| Application requesting access

| Authorization Server
| Keycloak

| Resource Server
| Protected API or application that accepts Access Tokens

| Access Token
| Credential presented to the API
|===

=== Client Credentials Flow

* Machine-to-machine access
* No browser
* No human user
* Client authenticates directly to Keycloak
* API receives an Access Token

=== PKCE

Proof Key for Code Exchange

. Application creates a random secret called the `code_verifier`
. It hashes that secret into a `code_challenge`
. Browser redirects to Keycloak with the `code_challenge`
. Application exchanges the authorization code for tokens and sends the original `code_verifier`
. Keycloak hashes the verifier and checks that it matches the earlier challenge

[.small]
Spec: https://www.rfc-editor.org/rfc/rfc7636[RFC 7636: PKCE]

=== Authorization Code Flow

* Browser-based login
* User authenticates with Keycloak
* Client receives an authorization code
* Client exchanges the code for tokens
* PKCE protects the code exchange

=== Device Authorization Flow

* Used by TVs, game consoles, CLIs, and appliances
* Device shows a short code and verification URL
* User signs in on a phone or laptop
* Device polls until authorization is complete

=== Password Flow Caution

* Legacy fallback
* Application collects the user's password
* Weak fit for MFA, SSO, federation, and passwordless
* Prefer Authorization Code + PKCE as previously discussed.
