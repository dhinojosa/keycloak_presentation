=== Keycloak

[.column]
--
image::keycloak-logo.png[width=75%]
--

[.column]
--
* Open source identity and access management
* Login once, reuse trust
* Centralize users, clients, roles, sessions, and tokens
* Speak standards: OpenID Connect, OAuth 2.0, SAML 2.0
--

=== What Keycloak Owns

* Authentication
* Token issuance
* Single sign-on
* User federation
* Identity brokering
* Authorization policies

=== What Keycloak Does Not Own

* Your business data
* Your employee source of truth
* Your application permission model
* Your secrets lifecycle
* Your production operations

=== Operational Responsibilities

[cols="35b,65"]
|===
| System | Responsibility

| Keycloak
| Authenticates users, manages SSO, and issues tokens

| Application
| Starts login, handles redirects, and creates its own session

| API
| Validates Access Tokens locally with JWKS

| Browser
| Carries redirects, authorization codes, and cookies

| Developer
| Configures clients, redirect URIs, roles, token validation, cookies, and logout
|===

=== Keycloak Concepts

[cols="30b,70"]
|===
| Concept | Meaning

| Realm
| Security boundary for users, clients, roles, and sessions

| Client
| Application or service that asks Keycloak for tokens

| User
| Person or account that authenticates

| Group
| Collection of users for shared assignment

| Role
| Named permission or authority

| Identity Provider
| External login source, such as Google, GitHub, LDAP, or SAML

| Session
| Login state tracked by Keycloak
|===

=== First Keycloak Setup

[cols="35b,65"]
|===
| Setting | Why it matters

| Realm
| Security boundary for this environment or tenant

| Client type
| Browser login, API bearer tokens, or machine-to-machine access

| Redirect URIs
| Where Keycloak may send authorization codes

| Web origins / CORS
| Which browser origins may call across domains

| PKCE
| Protects Authorization Code exchange

| Roles / scopes
| Access model exposed to applications

| Token lifetimes
| Controls stolen-token blast radius
|===
