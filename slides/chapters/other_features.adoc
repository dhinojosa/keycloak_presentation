== User Lifecycle Management
image::iStock-1181695869.jpg[background, size=cover]

=== User Lifecycle Management

* Provision for new users
* Update user information
* Assign roles and groups
* Disable or delete accounts
* Synchronize identities from external systems

=== Sources of Identity

* Local users managed in Keycloak
* LDAP / Active Directory synchronization
* Identity Federation (Google, GitHub, Microsoft)
* SCIM provisioning, if supported by your ecosystem
* Centralized identity management

=== Joiner, Mover, Leaver

* Identity and permissions evolve over time
* Joiner - employee joins the organization
* Mover - employee changes department or responsibilities
* Leaver - employee leaves the organization
* Keycloak centralizes these identity changes

== Fine-Grained Authorization
image::iStock-1181695869.jpg[background, size=cover]

=== Fine-Grained Authorization

* Goes beyond simple role-based access control (RBAC)
* Authorization can depend on users, groups, resources, or attributes
* Policies determine whether an operation is permitted
* Keycloak can evaluate authorization decisions centrally
* Useful for complex business and enterprise security rules

=== Fine-Grained Authorization Model

[cols="30b,70"]
|===
| Keycloak Concept | Example

| Resource
| `invoice:123`

| Scope
| `view`, `approve`, `pay`

| Policy
| Manager role, Finance department, invoice owner

| Permission
| Managers in Finance can approve invoices they own
|===

=== How Keycloak Does This

* Define protected resources
* Define scopes for actions
* Write policies from roles, groups, users, or attributes
* Attach policies to permissions
* Web Application asks Keycloak for an authorization decision

== I Don't Use Spring or Quarkus
image::iStock-1181695869.jpg[background, size=cover]

=== DIY Means You Own It

* Callback validation
* Token validation
* Token storage
* Logout behavior
* Role and claim mapping
* Error handling when Keycloak is unavailable

WARNING: Frameworks hide complexity. Custom auth code inherits it.

=== Checklist for DIY

* Download OIDC discovery metadata
* Use Authorization Code + PKCE
* Validate `state` on callback
* Refresh JWKS when `kid` is unknown
* Validate signature, issuer, audience, and expiration
* Protect Refresh Tokens
* Clear the local session and redirect on logout
* Map claims into local permissions deliberately
