== Spring
image::iStock-1181695869.jpg[background, size=cover]

=== Spring OAuth2 Login

* Protects browser pages
* Redirects users to Keycloak
* Receives an authorization code
* Exchanges the code for tokens
* Creates an application session
* Maps Keycloak roles into Spring authorities

=== Spring OAuth2 Client Configuration

[source,properties]
----
spring.security.oauth2.client.registration.keycloak.client-id=spring-app
spring.security.oauth2.client.registration.keycloak.authorization-grant-type=authorization_code
spring.security.oauth2.client.registration.keycloak.scope=openid,profile,email
spring.security.oauth2.client.provider.keycloak.issuer-uri=
  https://keycloak.example.com/realms/keycloak-demo
----

=== Spring Role Check

[source,java]
----
@GetMapping("/admin")
@PreAuthorize("hasRole('admin')")
String admin() {
  return "Hello Admin";
}
----

== Quarkus
image::iStock-1181695869.jpg[background, size=cover]

=== Quarkus OIDC Web Application

* Protects browser pages
* Redirects users to Keycloak
* Receives an authorization code on a callback path
* Creates an application session
* Maps Keycloak roles into Quarkus authorization

[source,properties]
----
quarkus.oidc.application-type=web-app
quarkus.oidc.auth-server-url=
  https://keycloak.example.com/realms/keycloak-demo
quarkus.oidc.client-id=quarkus-app
quarkus.oidc.authentication.redirect-path=/oidc/callback
quarkus.oidc.roles.role-claim-path=realm_access/roles
----

=== Same Role, Different Frameworks

* Both redirect users to Keycloak
* Both receive an authorization code
* Both exchange the code for tokens
* Both create an application session
* Both map Keycloak roles into framework authorization
