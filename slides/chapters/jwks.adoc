== JSON Web Key Set
image::iStock-1181695869.jpg[background, size=cover]

=== JSON Web Key Set (JWKS)

* Published set of public keys
* Used to verify JWT signatures
* Downloaded by applications and APIs
* Cached locally
* Refreshed when keys change

[.small]
Spec: https://www.rfc-editor.org/rfc/rfc7517[RFC 7517: JSON Web Key]

=== Key ID (`kid`)

* JWT header names the signing key
* `kid` means Key ID
* Verifier finds the matching key in JWKS
* Unknown `kid` usually triggers a JWKS refresh

[source,json]
----
{
  "alg": "RS256",
  "kid": "42",
  "typ": "JWT"
}
----

[.small]
Spec: https://www.rfc-editor.org/rfc/rfc7515#section-4.1.4[RFC 7515: `kid` Header Parameter]

=== Local Verification

* API receives a JWT
* API reads the `kid`
* API finds the matching public key
* API verifies the signature locally
* Keycloak is not called on every request
* Introspection is for opaque tokens or special revocation needs

NOTE: This is the API's job. Keycloak issued the token; the API must still validate it.

=== Passport Analogy

[cols="1,2", options="header"]
|===
| Thing | Analogy

| Keycloak
| The passport office.

| Private signing key
| The official seal/stamp that only the passport office possesses.

| JWT
| Your passport.

| JWKS
| The official specimen book that border agents use to recognize authentic passport seals.
|===
