== OpenID Connect
image::iStock-1181695869.jpg[background, size=cover]

=== OpenID Connect (OIDC)

* Adds identity and authentication to OAuth 2.0
* Defines who signed in
* Adds an ID Token
* Defines standard identity claims
* Publishes configuration through discovery

[.small]
Spec: https://openid.net/specs/openid-connect-core-1_0.html[OpenID Connect Core 1.0]

=== Three Tokens, Three Jobs

[cols="28b,47,25"]
|===
| Token | Job | Format

| Access Token
| Calls an endpoint that requires it
| Often a JWT in Keycloak

| ID Token
| Authentication result for the Web Application
| Always a JWT

| Refresh Token
| Requests new tokens from Keycloak
| Treat as opaque
|===

=== Inside an ID Token

[source,json]
----
{
  "iss": "https://identity.example.com/realms/workshop",
  "sub": "8f25c7b1-...",
  "aud": "reports-web",
  "exp": 1752188400,
  "iat": 1752188100,
  "auth_time": 1752188090,
  "nonce": "n-0S6_WzA2Mj"
}
----

NOTE: The ID Token identifies the authenticated user to the Web Application. It is not sent to Backend Services for access.

=== Authorization Code and State

[cols="35b,65"]
|===
| Value | Job

| Authorization Code
| Short-lived, one-time value exchanged for tokens

| `state`
| Random value created for each login and checked on callback

| `nonce`
| Random value checked inside the validated ID Token
|===

[source,plaintext]
----
/login/oauth2/code/keycloak?code=SplxlOBeZQQYbYS6WxSbIA&state=af0ifjsldkj
----

=== Proof Key for Code Exchange (PKCE)

. Web Application creates a random `code_verifier`
. It hashes that value into a `code_challenge`
. Browser redirects to Keycloak with the `code_challenge`
. Web Application exchanges the code with the original `code_verifier`
. Keycloak hashes the verifier and checks the earlier challenge

[.small]
Spec: https://www.rfc-editor.org/rfc/rfc7636[RFC 7636: PKCE]

[.notes]
--
PKCE protects the authorization code. It does not replace state, redirect URI
validation, HTTPS, cookies, or Web Application authentication to Keycloak. It is
required when a native or browser-only application cannot keep a secret and recommended for
server-side Web Applications.
--

=== UserInfo

* Optional source of additional profile claims
* Web Application sends an Access Token to Keycloak
* Keycloak returns claims allowed by granted scopes
* Not required to prove that authentication succeeded

[source,json]
----
{
  "sub": "8f25c7b1-...",
  "name": "Ada Lovelace",
  "email": "ada@example.com"
}
----

=== OIDC Discovery

* OIDC library retrieves and caches issuer metadata
* Metadata provides Keycloak endpoint locations
* `jwks_uri` identifies the public-key document
* Exact refresh timing depends on the library

[source,plaintext]
----
{issuer}/.well-known/openid-configuration
----

[source,json]
----
{
  "authorization_endpoint": "{issuer}/protocol/openid-connect/auth",
  "token_endpoint": "{issuer}/protocol/openid-connect/token",
  "userinfo_endpoint": "{issuer}/protocol/openid-connect/userinfo",
  "jwks_uri": "{issuer}/protocol/openid-connect/certs"
}
----

[.small]
Spec: https://openid.net/specs/openid-connect-discovery-1_0.html[OpenID Connect Discovery 1.0]

=== OIDC Callback Safety

* Validate `state`
* Validate the ID Token and `nonce`
* Use PKCE
* Lock down Redirect URIs
* Use HTTPS in production
